理解JavaScript回调函数并使用它们

发表于 2017-05-24 | 分类于 -日志 -JavaScript | | 阅读次数

在JavaScript中，函数是内置的类对象; 也就是说，函数是Object类型，它们可以像任
何其他对象（String，Array，Number等）一样使用，因为它们实际上是对象本身。它们可以“存储在变量中，作为参数传递给函数，在函数内创建，并从函数返回结果值” 。

因为函数是内置的类对象，所以我们可以将一个函数作为一个参数传递给另一个函数，
然后再执行那个传入函数，甚至返回到稍后执行。这是在JavaScript中使用回调函数的本质。在本文的其余部分，我们将学习关于JavaScript回调函数的一切。回调函数可能是JavaScript中最广泛使用的功能编程技术，您可以在几乎每一个JavaScript和jQuery代码中找到它们，但是它们对许多JavaScript开发人员仍然是神秘的。当你完成阅读这篇文章的时候，这个奥秘就不会再有了。

回调函数派生自称为功能编程的编程范例。在一个基本层面上，功能性编程规定了使用函数作为参数。功能性编程虽然在今天仍然在很小程度上仍然是一个深受专业训练的硕士程序员的深奥技术​​。

幸运的是，功能编程的技术已经被阐明，所以像你和我这样的凡人可以轻松理解和使用它们。功能编程的主要技术之一就是回调函数。正如你将要阅读的那样，实现回调函数与传递常量变量一样简单。这种技术很简单，我不知道为什么它主要涵盖在高级JavaScript主题中。

什么是回调或高阶函数？

回调函数也被称为高阶函数，是一个传递给另一个函数的函数（我们称之为另一个函数“otherFunction”）作为参数，并且回调函数在otherFunction中被调用（或执行）。回调函数本质上是一种模式（一种解决常见问题的模式），因此回调函数的使用也称为回调模式。

回调函数如何工作？

我们可以像使用变量一样使用函数，作为另一个函数的参数，在另一个函数中作为返回结果，在另一个函数中调用它。当我们将回调函数作为参数传递给另一个函数时，我们只传递函数定义,并没有在参数中执行该功能。换句话说，我们不像在执行一个函数时那样执行括号（）的拖尾对，而是传递函数。

当包含(调用)函数拥有了在参数中定义的回调函数后，它可以在任何时候调用(也就是回调)它。

请注意，回调函数不会立即执行，而是在包含函数的函数体内指定的位置“回调”它(形如其名)。所以，即使第一个jQuery示例如下所示：

匿名函数将延迟在click函数的函数体内被调用，即使没有名称，也可以被包含函数通过 arguments对象访问。

回调函数是闭包的
当作为参数传递一个回调函数给另一个函数时，回调函数将在包含函数函数体内的某个位置被执行，就像回调函数在包含函数的函数体内定义一样。这意味着回调函数是闭包的，想更多地了解闭包，请参考作者另一个贴子Understand JavaScript Closures With Ease。从所周知，闭包函数可以访问包含函数的作用域，所以，回调函数可以访问包含函数的变量，甚至是全局变量。

制作您自己的回调函数

现在你已经完全理解(我相信你已经理解了，如果没有请快速重新阅读一遍)了JavaScript关于回调的所用特性并且看到回调的使用是如此简单但功能却很强大。你应该看看自己的代码是否有机会使用回调函数，有以下需求时你可以考虑使用回调：

  • 避免重复代码 (DRY—Do Not Repeat Yourself)
  • 实现更好的抽象，您可以拥有更多通用功能（可以处理各种功能）
  • 具有更好的维护性
  • 增强代码的可读性
  • 有更多定制的功能
